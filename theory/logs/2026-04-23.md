### 정답/피드백

1. HTTP는 요청과 응답 메시지의 의미와 형식을 정합니다. TCP는 두 프로세스 사이에서 데이터를 안정적으로 전달하는 연결을 제공합니다.
2. 서버 프로세스가 3000번 포트로 들어오는 연결 요청을 받을 준비가 되어 있다는 뜻입니다.
3. 요청 body를 파싱하는 middleware 또는 body parser 설정이 빠졌는지 확인합니다.
4. HTTP 라우팅 문제에 더 가깝습니다. TCP 연결이 완전히 실패했다면 HTTP status code 자체를 받기 어렵습니다.
