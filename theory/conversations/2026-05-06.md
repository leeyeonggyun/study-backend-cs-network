## 상황 제시

너는 `API Gateway(Nginx) -> User-Service(Spring Boot)` 구조를 운영 중이야.  
오늘 09:10부터 알림이 들어왔고, 증상은 아래와 같아:

- **증상**
  - `/login`, `/profile` 요청의 **p95 latency**: `80ms -> 4.8s`
  - **5xx 비율**: `0.2% -> 7%`
  - 일부 클라이언트는 `upstream timed out` 응답 받음
- **트래픽**
  - RPS는 평소 대비 +15% (폭증은 아님)
- **인프라**
  - User-Service Pod CPU: 45~55% (높지 않음)
  - Memory 정상, GC pause도 평소와 유사
- **Nginx 에러 로그 일부**
  - `connect() failed (111: Connection refused) while connecting to upstream`
  - `upstream timed out (110: Connection timed out) while reading response header from upstream`
- **노드 네트워크 지표**
  - `SYN-SENT` 소켓 수 급증
  - `ESTABLISHED`는 큰 변화 없음
  - 재전송률(TCP retransmission) 상승
- **어제 변경사항**
  - User-Service replica 8 -> 16으로 스케일아웃
  - readinessProbe timeout 값을 짧게 조정
  - Nginx upstream keepalive 설정 일부 수정
