# 오늘의 신규 학습 주제

- 메인 주제: 컴퓨터, 프로그램, 프로세스 기초
- 이 주제를 고른 이유: 백엔드, 네트워크, CS를 이해하려면 먼저 "컴퓨터가 무엇을 실행하고 있는지"부터 알아야 한다.
- 이전 학습과의 연결: 오늘은 사실상 새 출발이다. 이후 클라이언트/서버, 포트, HTTP도 모두 이 위에 올라간다.
- 오늘 배워야 할 핵심 개념:
  - 컴퓨터는 프로그램을 실행한다.
  - 실행 파일/스크립트와 실행 중인 상태는 다르다.
  - 프로세스는 운영체제가 관리하는 실행 단위다.
- 실무에서 왜 중요한지: 서버가 안 뜬다, 프로세스가 죽었다, 포트를 누가 쓰고 있다 같은 말을 이해하려면 이 개념이 먼저 필요하다.

- 서브 주제: 입력과 출력, 터미널 명령의 의미
- 이 주제를 고른 이유: 프로그램은 입력을 받고 출력을 만든다. 이 감각이 있어야 요청/응답도 나중에 이해된다.
- 이전 학습과의 연결: 오늘 메인 주제의 "프로그램"이 실제로 무엇을 받으며 무엇을 내보내는지 보여준다.
- 오늘 배워야 할 핵심 개념:
  - 입력(input)과 출력(output)
  - 표준 입력(stdin), 표준 출력(stdout)이라는 기본 개념
  - 터미널 명령도 결국 프로그램 실행이라는 점
- 실무에서 왜 중요한지: 백엔드 프로그램 로그, 에러 메시지, 테스트 결과도 모두 출력이다.

# 오늘의 학습 설명

메인 주제 설명:
컴퓨터는 그냥 코드 파일을 "보관"만 하는 것이 아니라, 프로그램을 실제로 실행한다.  
예를 들어 `node hello.js`를 실행하면 `hello.js`라는 파일이 그대로 서버가 되는 게 아니라, Node라는 프로그램이 그 파일을 읽어 실행하고 운영체제는 그 실행 중인 상태를 하나의 프로세스로 관리한다.

자주 생기는 오해:
- 파일이 있으면 프로그램이 이미 돌고 있다고 생각하는 것
- 창이 안 보이면 실행 중이 아니라고 생각하는 것
- 서버를 매우 특별한 존재로 생각하는 것

실무 사용 예:
- 서버가 켜져 있지 않다 = 프로세스가 안 떠 있다
- 서버가 죽었다 = 프로세스가 종료됐다
- 재시작한다 = 프로세스를 다시 띄운다

트레이드오프/주의점:
- 오늘은 네트워크나 HTTP로 점프하지 않는다.
- 먼저 "실행된다 / 입력받는다 / 출력한다 / 살아 있다 / 종료된다"만 잡는다.

서브 주제 설명:
입력은 프로그램이 받는 값이고, 출력은 프로그램이 밖으로 내보내는 값이다.  
터미널에서 명령을 입력하면 프로그램이 실행되고, 그 결과가 텍스트로 출력된다.

실무 예시:
- `node hello.js`를 실행해 콘솔에 문장이 찍히면 그건 프로그램의 출력이다.
- `node echo-input.js`에 값을 입력해서 다시 돌려받으면 입력과 출력을 눈으로 확인한 것이다.
