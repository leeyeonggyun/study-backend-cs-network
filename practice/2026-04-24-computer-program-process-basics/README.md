# 프로그램과 프로세스 관찰 랩

## 목표

- 프로그램 파일과 실행 중인 프로세스의 차이를 이해한다.
- 입력과 출력이 무엇인지 직접 확인한다.
- 오래 살아 있는 프로세스가 어떤 느낌인지 본다.

## 실행 방법

### 1. 짧게 실행되는 프로그램

```bash
cd /Users/yeonggyun/study/study-system/practice/2026-04-24-computer-program-process-basics
npm run hello
```

관찰할 것:

- 텍스트가 출력된다.
- PID가 찍힌다.
- 출력 후 프로그램이 종료된다.

### 2. 입력을 받는 프로그램

```bash
npm run echo
```

관찰할 것:

- 내가 입력한 값이 다시 출력된다.
- 프로그램은 입력을 받을 때까지 기다린다.

### 3. 계속 살아 있는 프로세스

```bash
npm run process
```

관찰할 것:

- 2초마다 `heartbeat`가 출력된다.
- 이 프로그램은 자동으로 끝나지 않는다.
- `Ctrl+C`를 눌러 직접 종료해야 한다.

## 오늘 배워야 하는 포인트

- 파일은 저장된 코드다.
- 프로세스는 실행 중인 상태다.
- 출력은 프로그램이 밖으로 내보내는 텍스트다.
- 입력은 프로그램이 받는 값이다.

## 완료 체크리스트

- `npm run hello`를 실행했다.
- `npm run echo`로 입력/출력을 확인했다.
- `npm run process`를 실행하고 직접 종료했다.
- PID가 무엇인지 눈으로 확인했다.
